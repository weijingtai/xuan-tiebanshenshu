# 铁板神数模块产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品简介
铁板神数模块是一个专门用于中国传统命理学中铁板神数算法的计算引擎。该模块提供了多种传统算法的实现，包括六亲考刻法、定刻取数法、四柱天干取数法、日柱变卦取数法等，用于生成条文编号并进行命理分析。

### 1.2 产品定位
- **核心定位**：传统命理算法计算引擎
- **技术定位**：Flutter/Dart 实现的跨平台算法库
- **业务定位**：为命理应用提供准确、高效的铁板神数计算服务

### 1.3 产品价值
- 传承和数字化传统命理文化
- 提供标准化的铁板神数算法实现
- 支持多种算法策略，满足不同流派需求
- 为上层应用提供可靠的计算基础

## 2. 目标用户

### 2.1 主要用户群体
- **命理应用开发者**：需要集成铁板神数功能的应用开发团队

## 3. 功能需求

### 3.1 核心算法模块

#### 3.1.1 六亲考刻法 (SixQinCorrectKeCalculation)
**功能描述**：通过四柱干支计算先天基本卦和后天基本卦，结合变爻位置生成条文编号

**核心流程**：
1. 排四柱，取太玄数
2. 计算先天基本卦（年月干支配卦）
3. 计算后天基本卦（日时干支配卦）
4. 通过变爻和互卦生成四位基本数
5. 根据基本数计算条文编号列表

**输入参数**：
- FourZhu（四柱信息）
- gender（性别）

**输出结果**：
- 先天/后天基本卦
- 基本数
- 条文编号列表

#### 3.1.2 定刻取数法 (CorrectTimeAndKeCalculation)
**功能描述**：以被测者出生时刻为基准，查八刻数表确定基本数，转换为后天卦并进行纳甲配爻计算

**核心流程**：
1. 根据出生时刻查八刻数表
2. 确定基本数并转换为后天卦
3. 进行纳甲配爻计算
4. 计算卦总数和条文编号

**输入参数**：
- birthTimeZhi（出生时支）
- keInfoList（刻信息列表）

**输出结果**：
- 基本数和基本卦
- 六爻卦信息
- 条文编号列表

#### 3.1.3 四柱天干取数法 (FourZhuTianGanCalculation)
**功能描述**：仅使用四柱天干进行配数计算

**核心流程**：
1. 排四柱，只取天干
2. 按月、日、时、年顺序进行天干配数
3. 组成四位基本数
4. 递加96计算条文列表

#### 3.1.4 日柱变卦取数法 (DayGanZhiGuaCalculation)
**功能描述**：基于日柱干支进行卦象变换计算

**核心流程**：
1. 日支为上卦，日干为下卦组成基本卦
2. 计算互卦
3. 使用先天后天数组成四位基本数
4. 计算条文编号

### 3.2 高级算法模块

#### 3.2.1 四卦算法系列
- **四门法**：基于四柱的门户算法
- **八卦滚法**：八卦变换滚动算法
- **基础卦计算器**：支持多种卦象生成策略

#### 3.2.2 皇极算法系列
- **皇极一**：皇极数理第一种算法
- **皇极二**：皇极数理第二种算法
- **皇极三**：皇极数理第三种算法
- **皇极取数基础策略**：皇极算法的通用基础

#### 3.2.3 元堂算法
- **元堂计算器**：元堂派算法实现

### 3.3 数据模型

#### 3.3.1 核心数据结构
- **FourZhu**：四柱数据模型，包含年月日时干支及太玄数
- **SixYaoGua**：六爻卦数据模型，包含卦名、爻位、干支、六亲等
- **PureSixYaoGua**：纯六爻卦数据模型
- **KeInfo**：刻信息数据模型

#### 3.3.2 扩展功能
- **ChineseDateInfoExtension**：中国日期信息扩展，提供太玄数计算

#### 3.4.2 条文计算器

## 4. 非功能性需求

### 4.1 性能要求
- **计算响应时间**：单次算法计算 < 100ms
- **内存使用**：常驻内存 < 50MB
- **并发支持**：支持多线程并发计算

### 4.2 可靠性要求
- **算法准确性**：与传统手工计算结果100%一致
- **数据完整性**：确保所有常量映射表的完整性和准确性
- **异常处理**：对无效输入进行适当的异常处理

### 4.3 可维护性要求
- **代码结构**：采用策略模式，支持算法的灵活扩展
- **文档完整性**：每个算法都有详细的注释和文档
- **测试覆盖率**：核心算法测试覆盖率 > 90%

### 4.4 兼容性要求
- **平台支持**：支持 iOS、Android、Web、Desktop
- **Dart版本**：兼容 Dart 2.17+
- **Flutter版本**：兼容 Flutter 3.0+

## 5. 技术架构

### 5.1 整体架构
```
tiebanshenshu/
├── lib/
│   ├── constant/           # 常量定义
│   ├── domain/            # 数据模型
│   ├── service/           # 算法服务层
│   │   ├── classic/       # 重构后的面向对象算法
│   │   ├── four_gua/      # 四卦算法系列
│   │   ├── huang_ji/      # 皇极算法系列
│   │   ├── yuan_tang/     # 元堂算法
│   │   └── real_case_*.dart # 原始算法实现
│   ├── utils/             # 工具类
│   ├── ui/                # 用户界面
│   ├── providers/         # 状态管理
│   └── extensions/        # 扩展功能
```

### 5.2 设计模式
- **策略模式**：CalculationStrategy 抽象基类，支持多种算法策略
- **工厂模式**：算法实例创建和管理
- **建造者模式**：复杂数据结构的构建

### 5.3 数据流
1. 用户输入 → 参数验证 → 算法选择
2. 算法执行 → 结果计算 → 数据转换
3. 结果输出 → 界面展示 → 用户交互

## 8. 风险评估

### 8.1 技术风险
- **算法复杂性**：传统算法逻辑复杂，需要深入理解
- **数据准确性**：常量映射表的准确性直接影响计算结果
- **性能瓶颈**：复杂算法可能存在性能问题

### 8.2 业务风险
- **需求变更**：传统算法可能存在不同流派的差异
- **用户接受度**：数字化实现需要得到传统用户认可

### 8.3 风险缓解
- 建立完善的测试体系
- 与传统命理专家合作验证
- 采用模块化设计，降低变更影响
