# 元堂卦流运系统 PRD (Product Requirements Document)

## 1. 项目概述

### 1.1 项目目标
为元堂卦取数法增加完整的流运系统，包括大运卦、流年卦、流月卦的计算和可视化展示，使用户能够查看完整的命运运势走向及其卦象变化过程。

### 1.2 背景说明
元堂卦取数法目前已实现：
- ✅ 先天卦、后天卦的生成
- ✅ 大运期间的划分(6个大运，每个大运6-9年)
- ✅ 基础条文编号计算

**待实现的核心功能：**
- ❌ 至尊卦的特殊换卦规则
- ❌ 流年卦系统(每年一个流年卦)
- ❌ 流月卦系统(每年12个流月卦)
- ❌ 卦象变化过程的可视化展示

### 1.3 核心价值
1. **算法完整性**: 补全至尊卦换卦规则，确保核心算法准确性
2. **运势细化**: 从大运(6-9年) → 流年(1年) → 流月(1月)，三级运势体系
3. **变化追踪**: 可视化展示每一步卦象如何变化而来，增强用户理解
4. **实用价值**: 用户可以查询任意年龄、任意月份的运势卦象

## 2. 功能需求

### 2.1 至尊卦换卦规则 (Priority: P0 - 核心算法修正)

#### 2.1.1 业务规则
**至尊卦定义:**
- 坎为水(坎坎): 上卦坎 + 下卦坎
- 水雷屯(坎震): 上卦坎 + 下卦震
- 水山蹇(坎艮): 上卦坎 + 下卦艮

**触发条件:**
1. 先天命卦是至尊卦之一
2. 元堂爻在九五爻(索引4)或上六爻(索引5)

**换卦规则:**

| 元堂爻位 | 出生月 | 爻变规则 | 上下卦互换 | 示例(坎→后天卦) |
|---------|--------|----------|-----------|---------------|
| 九五(4) | 阴月   | 阳→阴    | ❌ 不互换 | 坎 → 师       |
| 九五(4) | 阳月   | 阳→阴    | ✅ 互换   | 坎 → 比       |
| 上六(5) | 阴月   | 阴→阳    | ✅ 互换   | 坎 → 井       |
| 上六(5) | 阳月   | 阴→阳    | ❌ 不互换 | 坎 → 涣       |

**月份判定:**
- 阳月: 寅(1)、辰(3)、午(5)、申(7)、戌(9)、子(11)
- 阴月: 卯(2)、巳(4)、未(6)、酉(8)、亥(10)、丑(12)
- 月份以节气为界,取自四柱的月支(干支历)

### 2.2 流年卦系统 (Priority: P0 - 核心功能)

#### 2.2.1 业务规则
**定义:** 每个虚岁年龄对应一个流年卦

**计算依据:**
1. 先确定该年龄所在的大运期(先天卦6个爻 or 后天卦6个爻)
2. 根据大运爻的阴阳性和大运初年的阴阳性,确定起卦方式
3. 逐年爻变,生成9个(阳爻大运)或6个(阴爻大运)流年卦

**阳爻大运(9年)的流年卦规则:**

| 大运初年 | 第1年卦 | 后续8年变换规则 |
|---------|---------|----------------|
| 阳年    | 先天卦/后天卦(不变) | 依次变换: (大运爻-2) → 大运爻 → (大运爻+1) → (大运爻+2) → 循环 |
| 阴年    | 先变换大运爻后为第1年 | 同上 |

**阴爻大运(6年)的流年卦规则:**

| 大运初年 | 第1年卦 | 后续5年变换规则 |
|---------|---------|----------------|
| 阳年/阴年 | 先变换大运爻后为第1年 | 依次变换: (大运爻+1) → (大运爻+2) → (大运爻+3) → ... |

**示例: 甲子年出生,先天卦元堂在上六(索引5)**
```
大运1: 上爻(阴爻,6年), 1-6岁
  - 大运初年: 甲子年(阳年)
  - 流年卦:
    1岁: 变换上爻 → 第1年卦
    2岁: 变换初爻(上爻+1) → 第2年卦
    3岁: 变换二爻(上爻+2) → 第3年卦
    ...
```

#### 2.2.2 数据结构
```dart
class YuanTangLiunianGua {

  final int yearIndex;        // 在大运中的年份索引(0-8或0-5)

  final String guaSource;     // 卦象来源("先天卦"/"后天卦")

  // 变化追踪
  final YuanTangDayunPeriod dayunPeriod; // 所属大运期
  final int changedYaoIndex;  // 本年变换的爻位(-1表示未变换)
  final String? previousGua;  // 上一年的卦象(第1年为null)

  const YuanTangLiunianGua({...});
}
```

#### 2.2.3 验收标准
- [ ] 能正确计算先天卦6个大运的所有流年卦(最多54年: 9+9+6+6+9+9)
- [ ] 能正确计算后天卦6个大运的所有流年卦
- [ ] 阳爻大运 + 阳年起算: 第1年不变换,后续按规则变换
- [ ] 阳爻大运 + 阴年起算: 第1年先变换大运爻,后续按规则变换
- [ ] 阴爻大运: 不论初年阴阳,第1年先变换大运爻,后续逐爻变换
- [ ] 所有流年卦的变化轨迹清晰可追踪

### 2.3 流月卦系统 (Priority: P1 - 增强功能)

#### 2.3.1 业务规则
**定义:** 每个流年有12个流月卦(1-12月)

**计算规则:**
1. **起卦基础:** 以该年的流年卦为本
2. **正月卦:** 变换(元堂爻-1)的爻位 → 正月卦
3. **阳月卦(1,3,5,7,9,11月):** 从正月卦开始,逐月向前变换下一爻
4. **阴月卦(2,4,6,8,10,12月):** 取对应阳月卦,变换其"应爻"

**应爻对应关系:**
```
初爻(0) ←→ 四爻(3)
二爻(1) ←→ 五爻(4)
三爻(2) ←→ 上爻(5)
```

**示例:**
```
假设流年卦为 "震坤", 元堂爻在三爻(索引2)

正月卦: 变换二爻(元堂爻-1) → 正月卦
二月卦: 正月卦变换其应爻(二爻的应爻=五爻) → 二月卦
三月卦: 正月卦变换三爻(二爻+1) → 三月卦
四月卦: 三月卦变换其应爻(三爻的应爻=上爻) → 四月卦
...
```

#### 2.3.2 数据结构
```dart
class YuanTangLiuyueGua {
  final int month;            // 月份(1-12)
  final bool isYangMonth;     // 月份阴阳
  final String gua;           // 流月卦象
  final int age;              // 所属年龄

  // 变化追踪
  final int changedYaoIndex;  // 本月变换的爻位
  final String? sourceGua;    // 源卦(阴月取自对应阳月)
  final int? yingYaoIndex;    // 应爻位置(阴月专用)

  const YuanTangLiuyueGua({...});
}
```

#### 2.3.3 验收标准
- [ ] 能正确计算任意流年的12个流月卦
- [ ] 阳月卦(6个): 正月起始,逐爻变换
- [ ] 阴月卦(6个): 从对应阳月卦的应爻变换而来
- [ ] 应爻对应关系正确(初↔四, 二↔五, 三↔上)
- [ ] 所有流月卦的变化来源清晰可追踪

### 2.4 卦象变化可视化 (Priority: P0 - 用户体验核心)

#### 2.4.1 功能描述
**需求:** 用户能够直观看到卦象如何一步步变化而来

**展示层级:**
1. **大运层:** 展示6个大运卦,标注爻位、年龄区间、阴阳性
2. **流年层:** 点击大运卦,展开该大运的所有流年卦,标注变换的爻
3. **流月层:** 点击流年卦,展开该流年的12个流月卦,标注变换来源

```
功能: 展示单个卦象及其变化信息

- 变爻高亮标注(红色/特殊样式)
- 变化箭头(从源卦 → 目标卦)
- 变化说明文字(如"变换五爻")
```

**组件2: DayunGuaList (大运卦展示列表)**
```
功能: 展示先天卦6个爻 + 后天卦6个爻,共12个大运期
布局: 垂直列表,每项显示:
  - 大运序号(1-12)
  - 爻位标签(初/二/三/四/五/上)

  - 年龄区间(如"1-9岁")
```

```
功能: 展示某个大运的所有流年卦(6-9个)

  - 年龄(虚岁)
  - 卦象

  - 点击展开流月卦
```

**组件4: LiuyueGuaPanel (流月卦展示面板)**
```
功能: 展示某个流年的12个流月卦

  - 月份(1-12)
```

#### 2.4.3 交互设计
```
用户操作流程:

4. 点击某个流年卦(如"3岁流年卦")

5. 每个卦象都显示:
   - 当前卦象的六爻图
   - 从上一个卦象如何变化而来(变换了哪一爻)
   - 变化的爻用特殊颜色标注
```

## 7. 风险与应对

### 7.1 算法复杂度风险
**风险:** 流年卦计算逻辑复杂,容易出错
**应对:**
- 严格按照文档规则实现
- 每个子功能独立测试
- 使用已知正确案例进行验证

### 7.2 性能风险
**风险:** 108个流年卦 + 1296个流月卦(108×12)计算量大
**应对:**
- 流年卦全量计算(用户打开页面时一次性计算)
- 流月卦懒加载(用户点击时才计算)
- 算法优化(避免重复计算)

### 7.3 UI复杂度风险
**风险:** 三层嵌套展示(大运→流年→流月)交互复杂
**应对:**
- 分阶段开发,先做大运和流年,再做流月
- 使用成熟的展开/折叠组件(ExpansionTile)
- 流月卦用对话框/底部Sheet独立展示,避免多层嵌套

| 阴年 | 天干为乙丁己辛癸的年份 |

### 9.3 示例数据

```
  流年卦:

  - ...
```
