# 铁板神数条文计算方案产品需求文档 (PRD)

## 1. 文档概述

### 1.1 文档信息
- **项目名称**: 铁板神数条文计算系统
- **文档版本**: v1.0
- **创建日期**: 2025-10-10
- **最后更新**: 2025-10-10

### 1.2 产品概述
本系统提供六种不同流派的铁板神数条文计算方案，包括：
1. **数据源展示** - 展示八字基础数据
2. **日干支卦** - 基于日柱干支变卦取数法
3. **四柱天干** - 基于四柱天干组合取数法
4. **太玄四柱** - 基于太玄纳甲取数法
5. **八卦加则** - 基于四柱装卦加则取数法（爻序法+纳甲法）
6. **元堂卦** - 基于元堂卦取数法（包含大运计算和条文扩展）

## 2. 业务背景

### 2.1 业务目标
为用户提供多种铁板神数条文计算方法，支持不同流派的取数算法，让用户可以对比不同算法的计算结果。

### 2.2 目标用户
- 铁板神数研究者
- 命理学爱好者
- 算法开发人员

### 2.3 使用场景
- 用户输入八字信息，系统自动计算并展示多种算法的条文结果
- 用户可以在不同算法之间切换对比
- 用户可以刷新重新计算结果

### 3.1 核心功能模块

#### 3.1.2 日干支卦计算模块
**功能描述**: 基于日柱干支，通过变卦取数法计算条文编号

**算法原理**:
1. 提取日柱的天干和地支
2. 天干配下卦，地支配上卦，组成基本卦（64卦之一）
3. 计算基本卦的互卦
4. 基本卦的上下卦取后天卦数（千位、百位）
5. 互卦的上下卦取先天卦数（十位、个位）
6. 组合成四位基础数
7. 应用条文计算配置生成条文列表（默认：基础数 ± 1000）

**详细需求**:
- 输入：日柱干支（JiaZi对象）
- 输出：基础数 + 条文列表
- 条文计算配置：
  - 标准配置：基础数 ± 1000
  - 简化配置：仅基础数
  - 扩展配置：基础数 ± 500、± 1000、± 2000

**用户交互**:
- 展示基础数计算过程
- 展示条文列表
- 支持下拉刷新
- 支持展开/收起详情

#### 3.1.3 四柱天干计算模块
**功能描述**: 基于四柱天干，通过天干配数法计算条文编号

**算法原理**:
1. 提取四柱天干（年干、月干、日干、时干）
2. 天干配数规则：甲1、乙6、丙2、丁7、戊3、己8、庚4、辛9、壬5、癸0
3. 按"月日时年"顺序排列，组合成四位基础数
4. 应用条文计算配置生成条文列表（默认：基础数 + 96*n，n=0-7）
5. 特殊处理：基础数插入条文列表首位

**详细需求**:
- 输入：八字四柱（EightChars对象）
- 输出：基础数 + 条文列表
- 条文计算配置：
  - 标准配置：基础数 + 96*n (n=0-7)，共8个条文
  - 简化配置：基础数 + 96*n (n=0-3)，共4个条文
  - 扩展配置：基础数 + 96*n (n=0-10)，共11个条文

**用户交互**:
- 展示四个天干的配数过程
- 展示基础数计算
- 展示条文列表（基础数在首位）
- 支持下拉刷新
- 支持展开/收起详情

#### 3.1.4 太玄四柱计算模块
**功能描述**: 基于四柱干支，通过太玄纳甲法计算条文编号，支持两种纳甲方案

**算法原理**:
1. 将四柱每柱的天干和地支分别配卦
   - 天干配卦：壬甲从乾数，乙癸向坤求，庚来震上里，辛在巽方留，己从离门起，戊以坎为头，丙须艮处出，丁向兑家收
   - 地支配卦：亥子坎宫寅木震，巳午离门丑在坤，卯酉乾金辰是兑，未申艮宫戌巽真
2. 组卦：同一柱的天干为上卦，地支为下卦
3. 纳甲配干支（两种方案）：
   - **方案A - 年干阴阳纳甲**：根据年干阴阳选择纳甲天干（阳年用 yangGuaYaoTianGan，阴年用 yinGuaYaoTianGan）
   - **方案B - 传统内外卦纳甲**：根据当前卦的内外卦阴阳选择纳甲天干（阳卦用 outerGuaYaoTianGan，阴卦用 innerGuaYaoTianGan）
   - 地支配置：两种方案都使用 innerGuaYaoDiZhi（下卦）和 outerGuaYaoDiZhi（上卦）
4. 太玄数计算：每爻的干支取太玄数相加（和为10则不计）
5. 上卦三爻数之和为上数，下卦三爻数之和为下数
6. 组合成基础数：上数*100 + 下数
7. 每柱产生2个基础数（年干阴阳纳甲1个 + 传统内外卦纳甲1个），共8个基础数
8. 应用条文计算配置生成条文列表（默认：各基础数 ± 96）

**详细需求**:
- 输入：八字四柱（EightChars对象）
- 输出：8个基础数（4柱 × 2方法）+ 对应的条文列表
- 纳甲方案：
  - **年干阴阳纳甲**：根据年干阴阳决定天干配置
  - **传统内外卦纳甲**：根据当前卦内外卦阴阳决定天干配置
- 条文计算配置：
  - 标准配置：基础数 ± 96
  - 简化配置：基础数
  - 扩展配置：基础数 ± 96、± 192
- 特点：
  - 两种纳甲方案同时显示
  - 多选框控制显示（默认都显示）
  - 每个结果携带详细的计算过程
  - 区分纳甲方案来源

**计算过程数据**:
- 柱名称：年柱/月柱/日柱/时柱
- 干支：如"甲子"
- 纳甲方案：年干阴阳/传统内外卦
- 上卦：卦名及后天数
- 下卦：卦名及后天数
- 六爻详情：
  - 爻位（初、二、三、四、五、上）
  - 纳甲天干
  - 纳甲地支
  - 天干太玄数
  - 地支太玄数
  - 是否被过滤（和为10）
- 上卦总和
- 下卦总和
- 最终基础数

**用户交互**:
- 多选框控制显示：
  - "年干阴阳纳甲"（默认勾选）
  - "传统内外卦纳甲"（默认勾选）
- 展示8个计算结果（分组显示）
- 每个结果显示：
  - 柱名 + 纳甲方案（如"年柱 - 年干阴阳纳甲"）
  - 基础数
  - 展开显示详细计算过程
  - 六爻详情可视化
- 展示每个基础数对应的条文列表
- 支持下拉刷新
- 支持展开/收起详情

#### 3.1.5 八卦加则计算模块
**功能描述**: 基于四柱干支，通过八卦装配地支加则法计算条文编号

**算法原理**:
1. 取四柱干支信息（年柱、月柱、日柱、时柱）
2. 根据天干地支配卦规则，将干支转化为八卦
   - 天干配卦：壬甲从乾数，乙癸向坤求，庚来震上里，辛在巽方留，己从离门起，戊以坎为头，丙须艮处出，丁向兑家收
   - 地支配卦：亥子坎宫寊木震，巳午离门丑在坤，卯酉乾金辰是兑，未申艮宫戌巽真
3. 组卦：天干为上卦，地支为下卦，得出四柱的四个卦象
4. 每卦装配六爻地支（两种方法）：
   - **方案A - 爻序法**：从上到下，阳爻依次配（子、寅、辰、午、申、戌），阴爻依次配（丑、卯、巳、未、酉、亥）
   - **方案B - 纳甲法**：使用传统六爻纳甲规则进行配置（不区分年干阴阳）
5. 将六爻每爻的地支配上对应的数字（yaoDiZhiNumberMapper），六个数相加得到总数
6. 计算条文数：上卦后天数（作为千位）+ 总数 - 下卦后天数
7. 每柱产生2个基础数（爻序法1个 + 纳甲法1个），四柱共8个基础数
8. **不扩展条文列表**：直接使用8个基础数作为条文编号

**详细需求**:
- 输入：八字四柱（EightChars对象）
- 输出：8个基础数（4柱 × 2方法），每个基础数即为条文编号
- 特点：
  - 每个基础数不扩展为条文列表
  - 每个结果携带详细的计算过程信息
  - 区分爻序法和纳甲法
  - 标注来源柱（年/月/日/时）

**计算过程数据**:
- 柱名称：年柱/月柱/日柱/时柱
- 干支：如"甲子"
- 算法方法：爻序法/纳甲法
- 上卦：卦名及后天数
- 下卦：卦名及后天数
- 六爻详情：
  - 爻位（初、二、三、四、五、上）
  - 阴阳性（阳/阴）
  - 配上的地支
  - 该地支的数字
- 六爻总和
- 计算公式：如"6000 + 270 - 2 = 6268"
- 最终条文数

**用户交互**:
- 展示8个计算结果（分组显示）
- 每个结果显示：
  - 柱名 + 算法方法（如"年柱 - 爻序法"）
  - 条文编号
  - 展开显示详细计算过程
  - 六爻卦象可视化
- 支持下拉刷新
- 支持展开/收起详情

#### 3.1.6 元堂卦计算模块
**功能描述**: 基于四柱干支，通过元堂卦取数法计算条文编号，包含大运计算和条文扩展功能

**算法原理**:
1. **步骤1：生成天地卦**
   - 提取四柱天干数列表
   - 提取四柱地支数列表（每个地支两个数）
   - 计算奇数总和 → 天数（模25处理）→ 天卦
   - 计算偶数总和 → 地数（模30处理）→ 地卦
   - 特殊处理：数为5时查询三元五宫映射表

2. **步骤2：生成上下卦（先天卦）**
   - 根据年份阴阳和性别决定上下卦位置
   - 阳年男性：天卦在上，地卦在下
   - 阳年女性：地卦在上，天卦在下
   - 阴年女性：天卦在上，地卦在下
   - 阴年男性：地卦在上，天卦在下

3. **步骤3：元堂装卦**
   - 判断时辰阴阳（阳时/阴时）
   - 根据时辰阴阳和爻的阴阳属性确定元堂爻
   - 根据阳爻/阴爻数量分三种情况装配地支：
     - 1-3爻：双重装配
     - 4-5爻：自上而下排列
     - 6爻全阳/全阴：三爻分组（考虑性别和节气）

4. **步骤4：生成后天卦**
   - 元堂爻爻变（阴→阳，阳→阴）
   - 上下卦互换

5. **步骤4.5：后天卦元堂装卦**
   - 后天卦也计算元堂爻位置
   - 使用与先天卦相同的装卦规则

6. **步骤5：计算互卦**
   - 计算先天卦互卦
   - 计算后天卦互卦

7. **步骤6：大运计算**
   - 从元堂爻开始循环6个爻位
   - 阳爻9年，阴爻6年
   - 先天卦大运（从1岁开始）
   - 后天卦大运（接着先天卦继续）

8. **步骤7：条文编号计算（8种方法）**
   - 加则法（先天卦、后天卦）
   - 纳甲太玄数法（先天卦、后天卦）
   - 本互法（先天卦、后天卦）
   - 互取数列表（先天卦、后天卦）

9. **步骤8：条文扩展**
   - 先天卦条文：基础数递加96四次（5个条文）
   - 后天卦条文：基础数递加96四次（5个条文）

**详细需求**:
- 输入：
  - 四柱信息（FourZhu对象）
  - 性别（"男" / "女"）
  - 三元（"上" / "中" / "下"）
  - 出生节气后（"夏至" / "冬至"）
- 输出：
  - 完整的中间计算过程（60+字段）
  - 先天卦和后天卦
  - 元堂爻信息（先天和后天）
  - 六爻地支配置
  - 大运列表（先天6个期间 + 后天6个期间）
  - 8种条文编号
  - 条文扩展列表（先天5个 + 后天5个）
- 条文扩展规则：
  - 先天卦：基础数 + [0, 96, 192, 288, 384]
  - 后天卦：基础数 + [0, 96, 192, 288, 384]
- 特点：
  - 保存完整的中间结果
  - 支持条文来源追溯
  - 每个条文标注来源算法
  - 展示大运年龄区间

**计算过程数据**:
- 步骤1数据：天干数、地支数、奇偶数和、天地卦
- 步骤2数据：年份阴阳、上下卦、先天卦
- 步骤3数据：时辰阴阳、六爻地支、元堂爻索引
- 步骤4数据：后天卦、爻变信息
- 步骤4.5数据：后天卦元堂爻、后天卦六爻地支
- 步骤5数据：先天卦互卦、后天卦互卦
- 步骤6数据：先天卦大运列表、后天卦大运列表

**用户交互**:

## 5. 技术架构

### 5.1 架构模式
采用Clean Architecture + MVVM模式：

```
Presentation Layer (UI)
├── Pages: StrategyDemoPage
├── ViewModels: DayGanZhiGuaViewModel, FourZhuTianGanViewModel, TaiXuanFourZhuViewModel, BaGuaJiaZeViewModel, YuanTangViewModel
└── Widgets: StrategyCard, LoadingWidget, ErrorWidget, YuanTangCard, YuanTangDayunWidget

Domain Layer (Business Logic)
├── UseCases: DayGanZhiGuaTiaoWenListUseCase, FourZhuTianGanTiaoWenListUseCase, TaiXuanFourZhuTiaoWenListUseCase, BaGuaJiaZeTiaoWenListUseCase, YuanTangTiaoWenListUseCase
├── Strategies: DayGanZhiGuaStrategy, FourZhuTianGanStrategy, TaiXuanFourZhuStrategy, BaGuaJiaZeStrategy, YuanTangStrategy
└── Models: BaseNumberModel, BaGuaJiaZeBaseNumberModel, YuanTangBaseNumberModel, MultiBaseNumberResult, TiaoWenListResult

Data Layer
└── Repository: TiaoWenRepository
```

### 5.2 设计模式
- **Strategy模式**: 封装不同的计算算法
- **Template Method模式**: UseCase基类定义通用流程
- **MVVM模式**: 分离UI和业务逻辑
- **Repository模式**: 抽象数据访问
- **Provider模式**: 状态管理

### 5.3 数据流
1. UI层触发计算 → ViewModel
2. ViewModel调用 → UseCase
3. UseCase调用 → Strategy（计算基础数）
4. UseCase处理 → 条文列表计算
5. UseCase查询 → Repository（获取条文数据）
6. UseCase返回 → MultiBaseNumberResult
7. ViewModel转换 → UITiaoWenListResultModel
8. UI层更新 → 展示结果

## 11. 附录

### 11.1 术语表
- **八字**: 出生年月日时的天干地支组合
- **干支**: 天干地支的简称
- **基础数**: 算法计算得到的初始数字
- **条文**: 铁板神数的断语文本
- **变卦**: 易经卦象的变化
- **纳甲**: 将天干地支配入卦爻的方法
- **太玄**: 太玄数取数理论
- **八卦加则**: 基于八卦装配地支的加则取数法
- **爻序法**: 按爻的阴阳性顺序装配地支的方法
- **六爻**: 卦象的六个爻位，从下到上为初、二、三、四、五、上
- **后天八卦数**: 后天八卦对应的数字（坎1、坤2、震3、巽4、乾6、兑7、艮8、离9）
- **元堂卦**: 基于元堂装卦理论的取数方法
- **元堂爻**: 根据时辰阴阳确定的特定爻位
- **天地卦**: 通过天干地支数值计算得到的初始八卦组合
- **先天卦**: 元堂卦算法中的初始卦象
- **后天卦**: 元堂爻爻变后上下卦互换得到的卦象
- **大运**: 人生各阶段的运势周期（阳爻9年，阴爻6年）
- **三元**: 上元、中元、下元，用于天地卦特殊情况处理
- **条文扩展**: 基础数递加特定偏移量得到多个条文编号
- **条文来源**: 标识条文编号来自哪个算法或方法
