# 卦中取数法 (GuaZhongStrategy) 算法审查

## 1. 算法概述

**名称**：卦中取数法（三种千位计算方案）
**类名**：`GuaZhongStrategy`
**文件路径**：`lib/service/strategy/gua_zhong_strategy.dart`
**描述**：基于四柱干支太玄数，分别生成年月卦和日时卦，每卦计算主卦和互卦的条文编号。由于千位计算存在"满10归零"导致位数不足的问题，该策略特别提供了三种千位计算方案并行输出。

## 2. 输入参数

该算法需要以下参数（通过 `GuaZhongStrategyParams` 传入）：

* `eightChars` (`EightChars`): 四柱信息。

## 3. 计算流程

1. **取太玄数**：
    * 获取四柱干支的太玄数 (`Constants.taiXuanGan/ZhiNumberMapper`)。
2. **年月卦计算**：
    * **上卦**：(年干太玄 + 年支太玄) % 8。
    * **下卦**：(月干太玄 + 月支太玄) % 8。
    * **生成主卦与互卦**。
    * **条文计算**：
        * `千位`: `(上卦先天数 + 6)` 处理结果（见下文方案）。
        * `百位`: 上卦先天数。
        * `十位`: 年干太玄数。
        * `个位`: 年支太玄数。
        * **互卦条文**：千百位同主卦，十位=互卦上卦先天数，个位=互卦下卦先天数。
3. **日时卦计算**：
    * **上卦**：(日干太玄 + 日支太玄) > 8 ? 减8 : 原值。
    * **下卦**：(时干太玄 + 时支太玄) > 8 ? 减8 : 原值。
    * **生成主卦与互卦**。
    * **条文计算**：
        * `千位`, `百位`: 同上。
        * `十位`: 日干太玄数。
        * `个位`: 日支太玄数。
4. **千位计算的三种方案**：
    * 当 `上卦先天数 + 6 == 10` 时（即上卦为震卦4）：
        * **方案1**（推荐）：取 `1`。`4+6=10 -> 1`，结果 `14xx`。
        * **方案2**：取先天数 `4`。`4+6=10 -> 4`，结果 `44xx`。
        * **方案3**：保留 `10`。`4+6=10 -> 10`，结果 `104xx`（五位数）。

## 4. 关键逻辑代码

```dart
// 千位计算方案
int _calculateQianWei_Plan1(int xiantianNumber) {
  final sum = xiantianNumber + 6;
  return sum == 10 ? 1 : sum % 10;
}
```

## 5. 依赖数据

* `Constants.taiXuanGan/ZhiNumberMapper`: 太玄数。
* `Constants.numberXianGuaMapper`: 数字 -> 先天卦。

## 6. 公式/资源管理

* **公式类型**：太玄数起卦 + 特定的千百十个位拼接公式。
* **资源文件**：无外部 JSON 依赖。

## 7. 审查结论

* **鲁棒性设计**：针对古代算法中模棱两可的边界情况（满10如何处理），代码层面直接实现了三种方案并行，交给用户或后续逻辑去选择，这是一种非常严谨的工程实践。
* **起卦法差异**：与 `QianHouGuaStrategy` 类似但有微小差异（日时卦取余逻辑不同：一个模8，一个减8），这种细节差异被准确实现。
* **输出丰富**：一次性输出12个条文编号，覆盖所有可能性。
