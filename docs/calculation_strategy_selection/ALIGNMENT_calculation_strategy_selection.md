# 计算策略选择功能对齐文档

## 项目上下文分析

### 现有项目架构
- **技术栈**: Flutter + Provider状态管理
- **架构模式**: MVVM + UseCase + Repository
- **核心组件**: 
  - Strategy层：负责基础数字计算
  - UseCase层：负责业务逻辑编排
  - ViewModel层：负责UI状态管理
  - Repository层：负责数据访问

### 现有计算策略体系
基于代码分析，当前系统已实现完整的条文列表计算框架：

#### 1. 基础数字计算（Strategy层）
- `TaiXuanFourZhuStrategy`: 太玄四柱算法
- `DayGanZhiGuaStrategy`: 日干支卦算法  
- `FourZhuTianGanStrategy`: 四柱天干算法

#### 2. 条文列表计算配置（TiaoWenListCalculationConfig）
当前支持三种计算方案：
- **loopAddTimes**: 基于基数和次数生成（如累加96 n次）
- **fromMultiples**: 基于倍数列表生成（如基数×[2,4,8,16]）
- **listAdd**: 自定义数字列表（如[96,192,384,768]）

#### 3. 现有配置方式
在`strategy_providers.dart`中硬编码配置：
```dart
Provider<TiaoWenListCalculationConfig>(
  create: (context) => TiaoWenListCalculationConfig.listAdd(
    customList: [96, 192, 384, 768],
    withSub: true,
  ),
),
```

### 现有UI界面
- 已实现`StrategyDemoPage`主页面
- 支持三个Strategy的展示和计算
- 使用卡片式布局展示不同算法
- 具备完整的状态管理和错误处理

## 需求理解确认

### 原始需求
用户希望：
1. 获得基础编号后可以选择不同的条文列表计算方案
2. 用户可以自行配置计算策略
3. 提供默认的计算方案
4. 在UI界面中展示不同计算策略对应的生成方案

### 边界确认
**任务范围**：
- 为现有三个Strategy添加计算策略选择功能
- 设计用户可配置的计算策略选择界面
- 保持现有架构不变，仅扩展功能
- 提供多种预设计算方案供用户选择

**不包含**：
- 修改现有Strategy的基础数字计算逻辑
- 重构现有的Repository或数据层
- 添加新的Strategy算法

### 需求理解
基于现有代码分析，我理解用户需求为：

1. **计算策略可选择性**：
   - 当前每个UseCase使用固定的`TiaoWenListCalculationConfig`
   - 需要支持用户在UI中选择不同的计算配置
   - 每种配置对应不同的条文生成方案

2. **预设方案提供**：
   - 提供多种常用的计算配置预设
   - 如：96累加法、48倍数法、自定义列表法等
   - 每种方案有清晰的描述和说明

3. **UI界面展示**：
   - 在现有Strategy卡片中添加计算策略选择器
   - 显示当前选择的计算方案描述
   - 展示不同方案对应的条文数量预览

## 疑问澄清

### 需要确认的关键决策点

1. **计算策略的作用域**：
   - 是否每个Strategy独立配置计算策略？
   - 还是全局统一的计算策略配置？
   - **建议**：每个Strategy独立配置，因为不同算法可能适合不同的计算方案

2. **预设方案的具体配置**：
   - 需要提供哪些具体的预设方案？
   - 基于现有代码，建议提供：
     - 96累加法（1-5次）
     - 48倍数法（×[2,4,8,16]）
     - 自定义列表法（[96,192,384,768]）
     - 简单累加法（基数+[1,2,3,4,5]）

3. **用户配置的持久化**：
   - 用户选择的计算策略是否需要保存？
   - **建议**：使用SharedPreferences保存用户偏好

4. **默认计算方案**：
   - 每个Strategy的默认计算方案是什么？
   - **建议**：保持当前配置作为默认方案

### 智能决策策略

基于现有项目架构和代码分析，我做出以下决策：

1. **架构扩展方案**：
   - 在ViewModel层添加计算策略选择逻辑
   - UseCase层支持动态传入计算配置
   - 保持Strategy层和Repository层不变

2. **UI设计方案**：
   - 在现有Strategy卡片中添加下拉选择器
   - 显示计算方案的描述和预期条文数量
   - 提供方案说明的详情对话框

3. **数据管理方案**：
   - 创建`CalculationStrategyConfig`管理类
   - 提供预设方案的工厂方法
   - 使用Provider管理全局配置状态

## 技术实现方案

2. **StrategyConfigSelector**: UI选择器组件

- 使用Provider管理策略配置状态
- 保持现有UseCase接口兼容性
