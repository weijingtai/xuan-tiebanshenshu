# 六亲考刻取数法 PRD（产品需求文档）

## 概述
- 目标：在不修改既有“皇极取数 v2”代码的前提下，新增“六亲考刻取数法”，并在架构与交互上保持贴近（可回滚、可跳步、可恢复）。
- 范围：仅 `tiebanshenshu` 子项目；新增代码放置在 `lib/features/liuqinkaoke`。
- 依赖：`DevConstant.dev_usa`（四柱八字/年干阴阳）、`PureSixYaoGua`（互卦/变爻）、常量映射表（太玄/先天/后天）、`TiaoWenRepository`（条文内容）。

## 用户故事
- 作为用户，我希望在同一页面同时选择“先天基本数”和“后天基本数”，每个提供 7 个候选（不变爻 + 初到六爻）。
- 作为用户，我希望候选卡片能清晰显示“变爻位”、“基本卦/互卦名称”以及条文摘要，便于比较。
- 作为用户，我希望提交时必须同时选中“先天”和“后天”，并在最终条文列表中看到每条条文的来源（先/后天、变爻位、±48n偏移）。
- 作为用户，我希望整个选择与计算过程支持回滚、跳步和恢复，避免重复操作。

## 功能范围
- 策略（Strategy）：仅进行数值计算与候选生成，不涉及 Session/Repository。
- 用例（UseCase）：编排流程、生成批量候选、批量拉取条文、校验与提交、计算最终条文列表。
- 视图模型（ViewModel）：驱动 UI 阶段流转、处理选择与提交、回滚与恢复。
- 页面（Page）：同页双栏展示（先天/后天），每栏 7 个候选；提交后展示最终条文列表。

## 关键逻辑
- 上/下卦确定：
  - 先天：依据“阳男阴女/阴男阳女”，用 `(年干太玄 + 年支太玄) % 8` 与 `(月干太玄 + 月支太玄) % 8` 决定上/下卦（映射至先天八卦）。
  - 后天：用 `(日干太玄 + 日支太玄 - 10) % 8` 与 `(时干太玄 + 时支太玄) % 8` 决定上/下卦（映射至后天八卦）。
- 互卦与变爻：使用 `PureSixYaoGua` 生成基本卦与互卦，遍历“0（不变爻）~6（六爻）”。
- 数字拼合：四位数由“基本卦上卦/下卦 + 互卦上卦/下卦”的先天或后天映射拼合；若结果 > 13000，统一折返 `- 12000`。
- 派生条文：在最终阶段，对选中的先天与后天基本数分别计算 `± (48 * n)`，n ∈ [2,4,8,16]，形成派生列表并拉取条文。
- 标注：候选与最终条文都必须标注来源（先天/后天）、变爻位、偏移。

## 数据保留约束
- 所有候选与派生数值必须保留原始值 `rawNumber`，并提供只读计算属性 `number`：当 `rawNumber > 13000` 时返回 `rawNumber - 12000`；否则返回 `rawNumber`。
- 该设计与 `DataBaseNumberDefinition` 保持一致；`rawNumber` 用于完整记录与回溯，`number` 用于 UI 展示与条文检索。
- 派生数（±48*n）同样保留其 `rawNumber` 与计算后的 `number`，以支持精确的来源标注与回滚。

## 非功能需求
- 性能：批量拉取候选条文，避免循环逐次 IO；尽量少改动既有代码。
- 可维护：与“皇极 v2”架构一致，文件组织清晰；策略与会话分离。
- 可靠：提交校验严格，必须同时完成先天与后天选择；支持回滚与恢复。

## 交互流程（阶段）
1. 初始化（initialized）：创建 Session，保存八字/年干阴阳/性别。
2. 候选准备（baseNumberSelectionReady）：UseCase 调用策略生成 14 个候选，并批量拉取条文，持久化为 `BaseNumberSelectionRecord`；UI 同页双栏展示。
3. 选择完成（baseNumberSelectionCompleted）：用户同时选中先天与后天；UseCase 校验并持久化。
4. 最终列表（finalTiaoWenListReady）：UseCase 计算派生数并拉取条文，页面按分组展示，支持回滚/跳步/恢复。

## 页面需求
- 选择页：双栏（先天/后天），各 7 个候选卡片；支持展开查看基本卦/互卦名称与变爻位；展示条文摘要；当两栏都已选择时，激活“提交”按钮。
- 结果页：展示最终条文列表；每条包含 `[先天|后天]·[变爻X]·[±48*n]` 标注；支持返回上一阶段重新选择。

## 数据结构约定
- 候选：`number`、`originKind（innate|acquired）`、`changeLineIndex（0~6）`、`baseGua`、`huGua`；不含正文内容（由 UseCase 回填）。
- 选择记录：复用 `BaseNumberSelectionRecord` 与分组机制，确保先后天两项同组、必须同时完成。
- 条文列表：在 `HuangJiTiaoWenNumber` 或相似模型中扩展元标注（或通过 `derivationChain` 描述）。

## 路由与依赖
- 路由：新增进入“六亲考刻取数法”选择页的路由，遵循现有 `navigator.dart` 路由风格。
- 依赖注入：在 `strategy_providers.dart` 或对应 Provider 里注册 UseCase/ViewModel/Repository，避免影响现有皇极功能。

## 验收标准
- 在同一页面展示 14 个候选，并能清晰区分先天/后天与变爻位。
- 提交必须同时包含先天和后天，校验与错误提示完整。
- 最终条文列表正确计算偏移，标注来源与变爻位完整。
- 所有新增代码位于 `tiebanshenshu/lib/features/liuqinkaoke`，未对现有代码造成破坏性修改。

## 迭代与扩展
- 后续可支持更多派生规则或自定义偏移集。
- 如果需要对候选模型增加字段，优先在新特性域内扩展，保持与“皇极 v2”兼容。
