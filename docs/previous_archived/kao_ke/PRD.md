## 1. 产品概述

### 1.1 背景

铁板神数是一种传统的命理推算方法,其中"考刻"是确定基础数的重要步骤。八刻秘数表考刻法通过让用户根据实际情况选择最符合的条文来确定"刻数"和基础数,是铁板神数中一种重要的起数方法。

### 1.2 产品目标

- 实现完整的八刻秘数表考刻交互流程
- 提供直观的表格式选择界面,支持用户快速定位和选择
- 支持两种后续计算方法(八卦加则、爻干支和数法)
- 支持会话管理和回滚功能,允许用户修正选择
- 提供详细的计算过程和结果展示

### 1.3 目标用户

- 铁板神数研究者
- 命理学爱好者
- 需要使用八刻秘数表起数的用户

## 2. 功能需求

### 2.1 核心功能流程

```
用户输入八字
  ↓
系统识别出生时辰
  ↓
展示该时辰对应的八刻秘数表(12时辰 × 8刻)
  ↓
用户选择最符合实际情况的条文
  ↓
系统记录选中的刻数和条文编号(基础数)
  ↓
系统执行卦象计算(前后两位数相加取模)
  ↓
用户选择后续计算方法(八卦加则 / 爻干支和数法)
  ↓
系统计算并展示最终条文结果
```

### 2.2 功能模块详细说明

#### 2.2.1 会话初始化

**功能描述**: 根据用户八字创建考刻会话

**输入**:
- 用户八字(EightChars)
- 会话名称(可选)

**输出**:
- 会话对象(KaoKeSession)
- 会话ID

**业务规则**:
- 自动提取出生时辰(DiZhi)
- 初始化会话状态为 `notStarted`
- 默认选择两种计算方法

---

#### 2.2.2 准备刻选择数据

**功能描述**: 加载并准备12时辰×8刻的完整数据供用户选择

**输入**:
- 会话对象
- KaoKeConstants (八刻秘数表常量)

**输出**:
- 12时辰的数据列表,每个时辰包含8刻数据
- 每刻包含: shiChen, ke, tiaoWenNumber, cipherText, originalText

**UI展示要求**:
- 使用Table格式展示
- 行: 12个时辰(子丑寅卯辰巳午未申酉戌亥)
- 列: 8个刻(初刻、初一刻、初二刻、初三刻、正刻、正一刻、正二刻、正三刻)
- 每个单元格显示: cipherText + tiaoWenNumber + originalText
- 用户出生时辰所在行使用黑色字体高亮,其他行使用灰色字体

**交互行为**:
- 用户点击任意单元格,弹出详情对话框
- 详情对话框展示完整的TiaoWenDataModel内容
- 用户确认选择后,记录该刻的信息

---

#### 2.2.3 提交用户选择

**功能描述**: 记录用户选择的刻数和条文编号

**输入**:
- 会话对象
- 选中的KaoEigthKeNumber

**输出**:
- 更新后的会话对象
- KeSelectionRecord (选择记录)

**业务规则**:
- 必须选择一个刻
- 记录选择时间戳
- 将条文编号作为基础数
- 更新会话阶段为 `keSelected`

---

#### 2.2.4 计算卦象

**功能描述**: 根据基础数(四位数)计算上下卦

**输入**:
- 基础数(tiaoWenNumber, 四位整数)

**计算规则**:
```dart
// 1. 分解四位数
int qian = baseNumber ~/ 100;  // 前两位
int hou = baseNumber % 100;     // 后两位

// 2. 计算上下卦数
int shangGuaNum = qian;
int xiaGuaNum = hou;

// 3. 大于8则对8取模
if (shangGuaNum > 8) {
  int remainder = shangGuaNum % 8;

  // 4. 如果余数==5 或 ==10, 则使用 (得数 - 10)
  if (remainder == 5 || remainder == 10) {
    shangGuaNum = shangGuaNum - 10;
  } else {
    shangGuaNum = remainder;
  }
}

// 下卦同理
if (xiaGuaNum > 8) {
  int remainder = xiaGuaNum % 8;
  if (remainder == 5 || remainder == 10) {
    xiaGuaNum = xiaGuaNum - 10;
  } else {
    xiaGuaNum = remainder;
  }
}

// 5. 将数值映射到后天八卦
// 1-乾, 2-兑, 3-离, 4-震, 5-巽, 6-坎, 7-艮, 8-坤
```

**输出**:
- GuaCalculationResult
  - shangGuaNumber: 上卦数值
  - xiaGuaNumber: 下卦数值
  - shangGuaName: 上卦名称
  - xiaGuaName: 下卦名称
  - fullGuaName: 完整卦名(如"天火同人")
  - calculationDetail: 计算详情文本

---

#### 2.2.5 计算最终条文

**功能描述**: 根据选择的计算方法,计算最终的条文结果

**输入**:
- 基础数
- 卦象计算结果
- 选择的计算方法集合

**支持的计算方法**:
1. **八卦加则** (`XianHoutianJiaZeStrategy`)
   - 使用先后天八卦加则进行计算

2. **爻干支和数法** (`LiuYaoGanZhiHeStrategy`)
   - 使用六爻干支和数法进行计算

**输出**:
- Map<KaoKeCalculationMethod, List<TiaoWenResult>>
- 按计算方法分组的条文结果列表

**UI展示要求**:
- 使用Tab或分组显示不同计算方法的结果
- 每个方法的结果独立展示
- 展示条文编号、内容、计算详情

---

#### 2.2.6 会话回滚

**功能描述**: 允许用户返回到之前的阶段重新选择

**支持的回滚点**:
- keSelectionReady: 回到选择刻之前
- keSelected: 回到选择刻之后,卦象计算之前
- baseNumberCalculated: 回到卦象计算之后,最终计算之前

**业务规则**:
- 每次阶段推进都自动创建快照
- 回滚时恢复快照状态
- 截断回滚点之后的历史记录

---

#### 2.2.7 计算方法选择器

**功能描述**: 允许用户选择需要使用的计算方法

**UI形式**: 多选Checkbox
- ☑ 八卦加则
- ☑ 爻干支和数法

**默认状态**: 两个都勾选

**交互行为**:
- 用户可以取消勾选
- 至少需要勾选一个方法
- 方法选择会影响最终计算结果

### 7.2 计算示例

**输入**: tiaoWenNumber = 6349

**步骤1**: 分解
- 前两位: 63
- 后两位: 49

**步骤2**: 取模运算
- 上卦: 63 % 8 = 7 (艮卦)
- 下卦: 49 % 8 = 1 (乾卦)

**结果**: 艮卦在上,乾卦在下 → 山天大畜卦
