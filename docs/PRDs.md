# Strategy演示 PRD

1. 背景与目标
- 在“UI主进入页面”提供一个清晰入口按钮“Strategy演示”，集中展示与对比本项目的关键数术策略与方法，支持输入用户基础信息后一键切换策略、查看过程解释与结果。
- 覆盖策略：
  1) 四门法 & 八卦滚法
  2) 皇极取数法 V2 新架构
  3) 六亲考刻取数法
  4) 八刻秘数考刻法
  5) 考订六亲

2. 范围与用户故事
- 角色：
  - 新手：快速体验各策略并理解关键步骤。
  - 研究者：比对不同策略的输出、可视化步骤与可导出数据。
  - 高级用户：自定义参数与规则，观察输出差异。
- 用户故事：
  - 作为用户，我在主页面点击“Strategy演示”进入演示页，输入出生信息后可在多个策略之间切换，并查看结果与详细步骤解释。
  - 作为研究者，我可以导出某策略计算过程与结果，用于复盘与论文引用。

3. 入口与导航设计（主页面按钮）
- 按钮文案：Strategy演示；位置：主页面右上角一级入口；图标：assets/icons/strategy（新建）；颜色：强调色（与主题一致）。
- 路由：/strategy_demo；导航：push -> StrategyDemoPage。
- 页面结构：
  - 顶部用户信息卡（出生日期、时间、地点、性别、校验状态）。
  - 策略切换Tab（五个策略）。
  - 结果展示区（核心数值/门位/卦象/六亲关系等）。
  - 过程解释区（步骤列表/滚动动画/规则说明）。
  - 操作区（参数设置、导出、复制链接）。
- 交互与状态：切换策略不丢失输入；参数变更即时计算（防抖）；最近一次参数缓存；错误提示与回退。

4. 通用输入与依赖
- 输入：公历/农历日期、时区、地点经纬（dataset/geo）、DST、性别、起局法选择；支持从“用户八字”复用（tiebanshenshu/lib/features/kao_ke/kao_ke_interactive_page.dart 注释：/// 用户八字）。
- 依赖：
  - 天文计算：web/assets/sweph.wasm + ephe；
  - 数据集：assets/dataset/geo；
  - 算法资源：assets/algorithms、assets/shen_sha；
  - 缓存：drift/sqlite3.wasm，最近参数与结果。
- 校验：日期范围、地点解析、时区合法性、空值处理、异常兜底文案。

5. V2 架构与通用接口
- 分层：Domain（核心算法与规则）→ UseCase（策略编排）→ Service/Presenter（参数与输出映射）→ UI（展示与交互）。
- 通用策略接口：
  - IStrategy { id, name, compute(input): StrategyResult, explain(): List<Step>, paramsSchema }
  - StrategyRegistry：注册/启用/禁用策略；可热插拔。
- 性能与可测试性：纯函数优先、可序列化输入/输出；计算耗时 < 100ms/策略（常规样例）。

6. 策略规格
6.1 四门法 & 八卦滚法
- 概述：根据四门定位与八卦序列滚动规则，推导门位、卦象及关键数。
- 输入：年/月/日/时、起局法、阴阳遁、盘型选择（定局/转局）。
- 输出：门位矩阵、卦象序列、滚法步骤（含每步依据）、关键数。
- UI：门位矩阵可视化、卦序滚动动画；步骤列表可展开。
- 校验：遇空/变卦/冲克的规则处理与说明。

6.2 皇极取数法 V2 新架构
- 概述：以皇极起数与飞布规则为核心，输出最终皇极数与过程解释；采用 V2 分层架构实现。
- 输入：皇极起数、节气界、阴阳遁、飞布与折返规则、容差设置。
- 输出：皇极数、取数轨迹、关键节点与理由；可导出 JSON/CSV。
- 性能：计算 < 100ms；缓存最近3次结果；单元测试覆盖率≥90%。

6.3 六亲考刻取数法
- 概述：基于八字与十神关系，考刻六亲并取数，输出关联度与定位。
- 输入：八字（年柱/月柱/日柱/时柱）、十神映射表、考刻规则集。
- 输出：六亲取数、关联度评分、位次与解释。
- UI：家系关系图谱（节点+评分条）；支持筛选与排序。
- 冲突处理：规则冲突时列出候选与理由，允许用户选择。

6.4 八刻秘数考刻法
- 概述：通过秘数库与映射规则，对八刻进行匹配与考刻，形成结果与置信度。
- 输入：八刻秘数库（本地/内置）、映射规则、校验项。
- 输出：秘数匹配结果、考刻路径、置信度评分与依据。
- UI：秘数分布图与路径展开；可复制路径说明。
- 合规：标注来源与版本；不合规项给出警示。

6.5 考订六亲
- 概述：结合用户八字与事件样本集，对六亲关系进行校订与报告。
- 输入：八字、事件样本（时间点/类别/结果）、校订算法（最大似然/贝叶斯可选）。
- 输出：校订建议、误差报告（偏差、可信区间）、规则修订提案。
- UI：对照表格、差异图、导出CSV。

7. 数据模型
- UserInput：日期、时间、地点、性别、起局法、策略参数。
- Bazi：年/月/日/时天干地支与十神关系。
- StrategyParams：各策略的参数集合与默认值。
- StrategyResult：核心数值/门位/卦象/评分/说明步骤 Step[]。
- Step：序号、依据、计算过程、中间值。
- CacheEntry：输入哈希、策略ID、结果、时间戳。

8. API 与集成
- 路由：/strategy_demo；页面组件：StrategyDemoPage。
- 服务：StrategyRegistry、StrategyService（compute、explain、export）。
- 缓存：Drift DAO（saveResult(inputHash, strategyId, result)）。
- 集成：
  - 复用 tiebanshenshu/kao_ke 的“用户八字”输入；
  - 使用 common/lib/helpers 与 ephe_io_helper（若可）获取天文与历法数据。

9. 验收标准
- 入口按钮可见、可点击，进入演示页；无崩溃。
- 五个策略均可计算并展示结果与步骤；切换策略不丢失输入。
- 常规样例计算时间 < 100ms/策略（在桌面与移动设备上）。
- 校验与错误提示完整；导出功能可用（JSON/CSV）。
- 单元测试覆盖率≥70%，关键算法≥90%。

10. 监控与埋点
- 埋点：页面进入、策略切换、计算耗时、导出事件、错误码。
- 指标：平均耗时、失败率、留存与使用频次。

11. 里程碑
- M1：入口与框架搭建，四门法&八卦滚法、皇极V2基础版。
- M2：完成六亲考刻取数法与可视化。
- M3：完成八刻秘数考刻法与导出。
- M4：完成考订六亲与报告模块；联调与优化。
- M5：性能与稳定性优化、文档完善与发布。

12. 风险与依赖
- 算法来源一致性与可验证性；数据集更新与合法性；时区与DST误差；低端设备性能；规则冲突处理的用户体验。

13. 术语
- 四门法：以四门定位进行推演的方法。
- 八卦滚法：按卦序规则进行滚动推算的方法。
- 皇极取数：依据皇极飞布等规则取数的体系。
- 六亲：父母、兄弟、子女等关联的十神关系。
- 八刻秘数：特定秘数与八刻的映射与推算。
