## 🔍 两种方案对比分析

### 核心差异

| 对比项 | 年干阴阳纳甲法 | 传统内外卦纳甲法 |
|-------|--------------|----------------|
| **天干配置依据** | 年干阴阳 | 内外卦位置 |
| **阳年/阳卦** | yangGuaYaoTianGan | outerGuaYaoTianGan |
| **阴年/阴卦** | yinGuaYaoTianGan | innerGuaYaoTianGan |
| **地支配置** | innerGuaYaoDiZhi + outerGuaYaoDiZhi | innerGuaYaoDiZhi + outerGuaYaoDiZhi |
| **适用场景** | 全局统一（由年干决定） | 逐卦独立（由卦位决定） |

### 相同点

1. ✅ 地支配置完全相同
2. ✅ 过滤规则相同（天干+地支=10则过滤）
3. ✅ 基础数计算公式相同（上卦和×100 + 下卦和）
4. ✅ 后天卦数映射相同

### 关键发现

1. **传统内外卦纳甲法更灵活**：
   - 内卦（下卦）固定使用 `innerGuaYaoTianGan`
   - 外卦（上卦）固定使用 `outerGuaYaoTianGan`
   - 符合传统六爻纳甲规则

2. **年干阴阳纳甲法更统一**：
   - 全局由年干阴阳统一决定
   - 计算逻辑更简洁
   - 四柱使用同一套天干配置

3. **两种方案产生不同结果**：
   - 同一八字，两种方案计算出不同的基础数
   - 验证了双方案架构的正确性

## 🐛 发现并修复的问题

### 问题 1: 传统内外卦纳甲法初始实现错误

**现象**:
- 第一次实现时，测试失败
- 预期：3342, 3326, 3945, 2648
- 实际：4242, 4226, 3945, 2639
- 只有日柱(3945)匹配

**原因**:
- 错误地根据每个卦的阴阳属性选择天干mapper
- 应该根据内外卦**位置**选择mapper，而非卦的阴阳

**修复**:
```dart
// ❌ 错误实现
final bool isLowerYangGua = Constants.yangGua.contains(zhiGua.name);
final bool isUpperYangGua = Constants.yangGua.contains(ganGua.name);

// ✅ 正确实现
final Map<Enum8Gua, List<TianGan>> lowerGanMapper = Constants.innerGuaYaoTianGan;
final Map<Enum8Gua, List<TianGan>> upperGanMapper = Constants.outerGuaYaoTianGan;
```

**验证**: 修复后所有测试通过 ✅
